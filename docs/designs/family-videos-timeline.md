## Vision

### 10x Check
The 10x version is the 12-month ideal: scene-split moments from each DVD, face recognition per person, and per-person timeline filtering. "Show me Sarah growing up" as a single gesture. Same architecture (manifest.json + static SPA), richer metadata. This is correctly an ocean, deferred.

### Landscape
No existing product combines self-hosted video + grandparent-simple + timeline browsing. FamilyAlbum, 23snaps, Cluster are cloud photo-first. Jellyfin/Plex handle self-hosted video but with complex UIs. The timeline concept is a genuine product insight. Clear build case.

## Accepted Scope — Implementation Notes

### 1. URL deep-linking (years + individual videos)
**Files:** `frontend/app.js`
**Behavior:** On page load, read `window.location.hash`. Disambiguation rule: if the hash is exactly 4 digits, treat as a year; otherwise, look up as a video ID in the manifest. If no video matches, fall back to year parsing. If `#YYYY` (e.g., `#1994`), set the timeline scrubber to that year. If `#video-id` (e.g., `#197902-198201-title00`), scroll to that video card and auto-open the player (this is the "text grandma a link" use case, she should land directly in playback). Update the hash when the user changes the selected year.
**Video IDs use the manifest `id` field** (e.g., `197902-198201-title00`), not human-readable slugs. Example: `videos.family.com/#197902-198201-title00`.
**Note:** The design doc's cookie-based passphrase gate was superseded by `auth_basic` per eng review decision #1.
**Auth interaction:** Verify that nginx `auth_basic` preserves URL fragments across the authentication flow. Test on iPad Safari specifically, as fragment handling varies across browsers.
**Schema:** No manifest changes needed. Uses existing `id` field.

### 2. DVD grouping UI with cover art
**Files:** `frontend/app.js`, `frontend/style.css`
**Behavior:** When rendering the video grid, group videos sharing the same `dvd` field. Render a group header with the DVD cover image (`cover` field) and a group title. Individual video cards appear as sub-items within the group. Single-title DVDs render as a regular card with the cover as a secondary element.
**Schema:** No manifest changes needed. The `dvd` and `cover` fields already exist in the manifest schema (see `docs/plan.md` lines 115-119).
**Mobile:** DVD groups render as stacked sections with no collapse; cover images display inline above the group's video cards in single-column layout.
**Note:** Metadata correctness for grouping depends on the `overrides.json` mechanism defined in the implementation plan.

### 3. Keyboard arrow navigation
**Files:** `frontend/app.js`
**Behavior:** Left/Right arrow keys move the timeline scrubber to the previous/next year. When a video card has focus (via Tab), Enter or Space opens it in the player overlay. Escape closes the player overlay. Focus management: after closing the player, return focus to the card that was playing.
**Scope:** Global keydown listener on `document`. Scrubber arrows only active when player overlay is closed.

### 4. Video count density on timeline
**Files:** `frontend/app.js`, `frontend/style.css`
**Behavior:** Below each year label on the timeline, render a proportional bar whose height represents the number of videos in that year (or a numeric badge showing the count). Computed from the manifest: count videos whose `dateStart`-`dateEnd` range includes that year.
**Schema:** No manifest changes. Computed client-side from existing date fields.

### 5. Lazy thumbnails + preconnect
**Files:** `frontend/app.js`
**Behavior:** Use `IntersectionObserver` to lazy-load thumbnail images. Cards outside the viewport use a placeholder; thumbnails load when scrolled into view. On video card hover (desktop) or tap-hold (mobile), add a `<link rel="preconnect">` hint for the video file URL to warm the connection before the user clicks play. Note: this is video file preconnection, not sprite-based hover preview (sprites are deferred to V2).
**Fallback:** If `IntersectionObserver` is unavailable (Safari < 12.2, very old iPads), load all thumbnails eagerly. Acceptable for ~50-100 videos.
**Schema:** No manifest changes.

## Date Range Note
The actual date range of the collection is determined by the manifest data at runtime. The design doc references 1989-2004 as an example; the implementation plan uses 1979-2006 based on real filenames. The frontend reads `dateRange` from `manifest.json` and does not hardcode either.
